```json
{
  "slug": "my-animation",

  "why": "This matters because...",
  "canva_url": "https://canva.com/design/...",
  "status": "storyboard",
  "created_at": "2026-07-09T10:00:00Z",
  "updated_at": "2026-07-09T10:30:00Z",
  "storyboard_prompts": {
    "act-1": "Image prompt used for act-1 storyboard...",
    "act-2": "Image prompt used for act-2 storyboard...",
    "act-3": "Image prompt used for act-3 storyboard..."
  },
  "acts": {
    "act-1": {
      "role": "problem",
      "outline": "done",
      "script": "done",
      "components": "done",
      "audio": "done"
    },
    "act-2": {
      "role": "solution",
      "outline": "done",
      "script": "done",
      "components": "done",
      "audio": "done"
    },
    "act-3": {
      "role": "lesson",
      "outline": "done",
      "script": "done",
      "components": "done",
      "audio": "done"
    }
  }

```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `slug` | string | URL-safe project identifier, derived from title |
| `title` | string | Display name |
| `topic` | string | Full topic description fed into AI prompts |
| `component_type` | string | Animation style: `explainer`, `tutorial`, `story`, `demo` |
| `question` | string | Pedagogical: what question does this answer? |
| `answer` | string | Pedagogical: the core answer/insight |
| `why` | string | Pedagogical: why this matters |
| `canva_url` | string | Link to Canva project (optional) |
| `status` | string | Current pipeline stage: `outline`, `script`, `components`, `audio`, `storyboard` |
| `created_at` | string | ISO 8601 timestamp |
| `updated_at` | string | ISO 8601 timestamp |
| `storyboard_prompts` | map | Per-act image prompts used in storyboard (feeds script re-generation) |
| `acts` | map | Per-act status map |

## Act Structure

Fixed 3-act narrative structure, defined in `server/acts.go` and mirrored in `shared/acts.py`:

| Key | Slug | Role | Title | Purpose |
|-----|------|------|-------|---------|
| `act-1` | `act-1-problem` | `problem` | Act 1 — Problem | Set up the world and the problem/pain the audience feels. |
| `act-2` | `act-2-solution` | `solution` | Act 2 — Solution | Introduce the solution; show how the problem is resolved. |
| `act-3` | `act-3-lesson` | `lesson` | Act 3 — Lesson | The takeaway / moral / insight the audience leaves with. |

Each act has independent status tracking:
```json
{
  "role": "problem",
  "outline": "done",      // "" | "done"
  "script": "done",       // "" | "done"
  "components": "done",   // "" | "done"
  "audio": "done"         // "" | "done"
}
```

## Outline (`outline.json`)

Project-level 3-act outline generated by OpenRouter:
```json
{
  "acts": {
    "act-1": {
      "summary": "Set up the world and the problem...",
      "title": "The Challenge"
    },
    "act-2": {
      "summary": "Introduce the solution...",
      "title": "Breaking Through"
    },
    "act-3": {
      "summary": "The key takeaway...",
      "title": "What We Learned"
    }
  }
}
```

## Script Beats (`<act-slug>/script/beats.json`)

Per-act structured script with narration and beat sequence:
```json
{
  "narration": "Full narration text for voiceover synthesis...",
  "beats": [
    {"id": "beat-1", "text": "Opening scene: establish the context..."},
    {"id": "beat-2", "text": "Introduce the key conflict or question..."},
    {"id": "beat-3", "text": "Show the resolution approach..."},
    {"id": "beat-4", "text": "Reveal the insight or takeaway..."}
  ]
}
```

Also stored as markdown: `<act-slug>/script/act.md`

## Components (`<act-slug>/components/components.json`)

Per-act typed component manifest:
```json
[
  {
    "id": "my-animation-background-01",
    "type": "background",
    "prompt": "Full image generation prompt...",
    "file": "act-1-problem/components/my-animation-background-01.png",
    "script_ref": "beat-1"
  },
  {
    "id": "my-animation-lower-third-01",
    "type": "lower-third",
    "prompt": "...",
    "file": "act-1-problem/components/my-animation-lower-third-01.png",
    "script_ref": "beat-2"
  }
]
```

### Component Types

9 built-in types, each with a defined visual role:

| Type | Role in Video | File Naming Pattern |
|------|--------------|---------------------|
| `background` | Full-frame scene backdrop for a beat | `<slug>-background-NN.png` |
| `lower-third` | Text/label bar overlay (names, captions, stats) | `<slug>-lower-third-NN.png` |
| `speech-bubble` | Dialogue / quote / thought callout | `<slug>-speech-bubble-NN.png` |
| `infographic` | Data / numbers / diagram visualizing a point | `<slug>-infographic-NN.png` |
| `character` | On-screen figure / subject / mascot | `<slug>-character-NN.png` |
| `icon` | Small symbolic graphic accent | `<slug>-icon-NN.png` |
| `title-card` | Full-screen heading for the act or a key line | `<slug>-title-card-NN.png` |
| `transition` | Animated connecting element between beats | `<slug>-transition-NN.png` |
| *(extensible)* | New types can be added without pipeline changes | |

## Sprite Generator (`sprite/sprite.json`)

Project-level 14-icon technical sprite sheet. Concepts are extracted from the
script via a text call; the image is generated from those concepts:

```json
{
  "concepts": [
    "Chaotic Spreadsheet", "Conflicting Data", "Disconnected Tools", "Central Hub",
    "Database Icon", "CRM Connector", "ERP Connector", "Digital Highway",
    "Data Chaos", "Data Stream", "Lightbulb Innovation", "Gear Adaptation",
    "Shield Resilience", "Bridge Of Light"
  ],
  "versions": [
    {
      "id": 1,
      "file": "sprite/sprite-sheet-01.png",
      "prompt": "Create a clean, technical sprite sheet image containing exactly 14...",
      "concepts": ["Chaotic Spreadsheet", "..."],
      "created_at": "2026-07-13T10:40:51Z"
    }
  ]
}
```

Versioned like storyboard images — monotonic IDs, never overwrites a prior
sheet. See [16-sprite-generator.md](16-sprite-generator.md) for the full spec.

## Audio

Per-act audio files:
```
<act-slug>/audio/

```

## Storyboard (`storyboard/`)

### `storyboard.json`
Scene-by-scene plan generated from scripts + components:
```json
{
  "act-1": {
    "scenes": [
      {
        "scene_id": "scene-1",
        "beat_ref": "beat-1",
        "components": ["my-animation-background-01", "my-animation-lower-third-01"],
        "narration": "...",
        "duration": "5s"
      }
    ]
  },
  "act-2": { ... },
  "act-3": { ... }
}
```

### `versions.json`
Versioned per-act image generation history:
```json
{
  "versions": [
    {
      "id": 1,
      "act": "act-1",
      "file": "storyboard/storyboard-act-1-01.png",
      "image_prompt": "The full image prompt used...",
      "image_model": "google/gemini-3-pro-image",
      "created_at": "2026-07-09T10:20:00Z"
    }
  ]
}
```

### Image files
```
storyboard/
├── storyboard.json
├── versions.json
├── storyboard-act-1-01.png    # act-1, version 1
├── storyboard-act-1-02.png    # act-1, version 2 (re-generation)
├── storyboard-act-2-01.png    # act-2, version 1
└── storyboard-act-3-01.png    # act-3, version 1
```

Versions are monotonic across all acts (never re-use IDs) and within each act (clean file numbering).

## Prompt Audit Trail (`<slug>/prompts/`)

Every generation call saves its full prompt for debugging:
```
prompts/
├── 20260709-102005-outline-outline.json
├── 20260709-102010-act-1-script.json
├── 20260709-102015-act-1-component-background.json
├── 20260709-102016-act-1-component-lower-third.json
├── 20260709-102020-act-1-audio.json
├── 20260709-102025-act-1-music.json
├── 20260709-102030-act-1-sfx-whoosh.json
└── 20260709-102035-storyboard-storyboard.json
```

Format:
```json
{
  "timestamp": "2026-07-09T10:20:05Z",
  "act": "act-1",
  "step": "component-background",
  "prompt": "Full rendered prompt text..."
}
```

For chat-based calls (outline, script, storyboard JSON), the prompt includes system + user messages:
```
[system]
You are a script writer...

[user]
Generate a script for {{topic}}...
```
